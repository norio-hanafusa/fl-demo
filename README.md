# 🏥 Federated Learning サンドボックス（病院版）

複数の病院が **患者データを一切外に出さない** まま、サーバーを介して 1 つの予測モデルを
共同で育てていく様子を、ブラウザだけで体感できる教育用デモです。

外部ライブラリ・CDN・サーバー通信は一切なく、すべてブラウザ内で計算します
（オフライン／`file://` でも動作）。GPU も不要です。

## 🔗 ライブデモ

**<https://norio-hanafusa.github.io/fl-demo/>**

ブラウザで開いてすぐに試せます。

## 使い方

### 1. すぐ試す（ライブデモ）

上記の <https://norio-hanafusa.github.io/fl-demo/> を開くだけです。インストール不要。

### 2. ローカルで開く（ダウンロード）

1. [index.html](index.html) を保存する（GitHub 上で `index.html` →「Download raw file」）
2. 保存した `index.html` をダブルクリックしてブラウザで開く

外部通信が無いので、ネットに繋がっていなくても動きます。

### 3. リポジトリごと取得する

```bash
git clone https://github.com/norio-hanafusa/fl-demo.git
cd fl-demo
# index.html をブラウザで開くだけ（ビルド不要）
```

### 動かし方（共通）

1. **「▶ 再生」** を押すと、ラウンドが自動で進みます（**「⏭ 1ラウンド」** で 1 サイクルずつ進めることも可能）
2. サーバー → 各病院へ「重み」が配られ、各病院が自院データだけで学習し、重みだけを返し、サーバーが集約する流れを順に追えます
3. 下部のグラフで、連合学習の精度が「単独学習の平均」を上回り「全データ集約の理想」へ近づくのを確認できます
4. スライダー（病院数 / 非 IID の強さ / ローカル学習回数 / 学習率）を変えて挙動を試せます
5. **「⟲ リセット」** でデータを作り直して最初から

> 💡 講演での実演にも、聴衆が触る探索にも使えます。**非 IID の強さ** を上げると「持ち寄らないと賢くなれない」ことが、下げると「単独でも追いつく」ことが見えます。

## 何が見えるか

1 ラウンド = 連合学習（Federated Learning）の 1 サイクルを、4 ステップを順に光らせて可視化します。

1. **配布** — サーバーが共有モデルの重み（数値）を各病院へ送る
2. **ローカル学習** — 各病院が *自院の患者データだけ* でモデルを微調整（データは院外に出ない）
3. **返送** — 各病院は学習後の *重みだけ* をサーバーへ返す
4. **集約（FedAvg）** — サーバーが重みを患者数で加重平均し、新しい共有モデルにする

## 「データが出ない」を裏づける仕掛け

- **通信ゼロ** — ソースに `fetch` / `XHR` が 1 つも無く、全計算がブラウザ内
- **プライバシー台帳** — 院外に出た「重み（バイト数）」と、院内に留まった「患者レコード（送信 0 回）」を対比表示
- **非 IID** — 病院ごとに患者層が偏っており、1 病院だけでは正しい境界を引けないことが散布図で分かる
- **精度グラフ** — 連合学習が「各病院の単独学習の平均」を上回り、「全データ集約の理想（上限）」へ接近していく

## 操作

- ▶再生 / ⏭1ラウンド / ⟲リセット、再生速度
- スライダー: 病院数 (2〜6) / 非 IID の強さ / ローカル学習回数 / 学習率
- 非 IID を強くすると連合学習の価値が際立ち、IID に近づけると単独学習でも追いつく

## モデルについて

- 2 次元特徴（例: 年齢・検査値）→ 2 値アウトカムを予測する **ロジスティック回帰**（重み 3 個）
- 決定境界が直線なので「モデルが賢くなる = 境界線が動く」のが直感的に見える
- データはすべて合成（架空）。実在の患者データは含みません

## ライブデモの公開について（GitHub Pages）

ライブデモは GitHub Pages で配信します。リポジトリの **Settings → Pages** で、
**Source: Deploy from a branch**、**Branch: `main` / `(root)`** を選んで保存すると、
数十秒〜数分で <https://norio-hanafusa.github.io/fl-demo/> が有効になります。
